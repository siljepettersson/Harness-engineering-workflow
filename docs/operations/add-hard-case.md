# Add Hard Case

Use this workflow when a failure should be preserved.

1. Record the original question.
2. Record the resolved question if conversation state was used.
3. Record expected sources, expected table, or expected facts.
4. Record the failure mode.
5. Add the case to the relevant file under `data/evals/cpi/`.
6. Mention the origin if it came from user feedback, lint, or regression.

Hard cases are regression assets.
